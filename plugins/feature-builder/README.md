# Feature Builder Plugin

기능 추가 요청 시 사용자의 의도를 체계적으로 파악하여 정확한 구현을 돕는 플러그인입니다.
프론트엔드/백엔드 공통 사용 가능합니다.

## 문제

```
❌ "로그인 기능 추가해줘"
   → AI가 추측해서 구현
   → 원하는 것과 다름
   → 수정 요청 반복
```

## 해결

```
✅ "로그인 기능 추가해줘"
   → 체계적인 질문으로 의도 파악
   → 명확한 스펙 정리
   → 정확한 구현
```

## Installation

```bash
/plugin marketplace add wigtn/wigtn-plugins-with-claude-code
/plugin install feature-builder
```

## Usage

```bash
/feature                    # 대화형 가이드 시작
"검색 기능 추가해줘"         # 자연어로 요청
"로그인 만들어줘"            # 자연어로 요청
```

│ Phase 4: 의도 정리 & 확인                   │

└─────────────────────────────────────────────┘

## 질문 최적화

### 1. 빠른 경로

이미 명확한 요청은 불필요한 질문을 건너뜁니다:

```
"이메일/비밀번호 로그인, JWT 발급, /auth/login API"
→ 엣지케이스 질문부터 시작
```

### 2. 점진적 상세화

한 번에 모든 질문을 하지 않습니다:

```
1차: WHAT, WHY, WHO
2차: WHERE, WHEN, HOW
3차: 엣지케이스, 제약사항
```

### 3. 예시 제공

```
"검색 기능이라고 하셨는데, 다음 중 어떤 것에 가까운가요?
• 단순 키워드 검색 (예: 구글)
• 필터링 검색 (예: 쿠팡)
• 실시간 자동완성 (예: YouTube)"
```

### 4. 기본값 제안

```
"다음과 같이 구현하면 될까요?
• 인증: JWT 기반
• 에러 처리: 토스트 메시지
• 로딩: 스켈레톤 UI"
```

## 일반적인 기능 템플릿

### 인증 기능

```
1. 인증 방식: 이메일+비번 / 소셜 / 전화번호 / 매직링크
2. 추가 기능: 회원가입 / 비번찾기 / 이메일인증 / 2FA
3. 세션 관리: JWT / 세션쿠키 / 멀티디바이스
```

### CRUD 기능

```
1. 기본 작업: List / Detail / Create / Update / Delete
2. 추가 기능: 검색 / 필터 / 정렬 / 페이지네이션 / 일괄작업
3. 권한: 작성자만 / 관리자 / 공개설정
```

### 검색 기능

```
1. 검색 방식: 키워드 / 자동완성 / 필터 / 전문검색
2. 검색 대상: 단일 리소스 / 통합 검색
3. UI 형태: 검색바 / 드롭다운 / 전용페이지 / 사이드바
```

### 파일 업로드

```
1. 파일 유형: 이미지 / 문서 / 동영상
2. 업로드 방식: 단일 / 다중 / 드래그앤드롭
3. 저장 위치: 로컬 / S3 / GCS
4. 추가 기능: 리사이징 / 썸네일 / 진행률
```
