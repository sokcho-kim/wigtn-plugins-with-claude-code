| ----------------- | --------------------------------------------------- |
| **컴포넌트 유형** | 버튼/입력폼/카드/모달/드롭다운/테이블/리스트?       |

## 공통 심층 질문

### 성능 관련

| 질문              | 세부 고려사항                              |
| ----------------- | ------------------------------------------ |
| 예상 트래픽은?    | 동시 사용자, 요청/초, 피크 시간            |
| 응답 시간 목표는? | 일반 요청, 무거운 쿼리, 업로드             |
| 캐싱 필요?        | 캐시 대상, TTL, 무효화 전략                |
| 최적화 포인트?    | DB 쿼리, N+1, 이미지 최적화, 코드 스플리팅 |

### 보안 관련

| 질문         | 세부 고려사항                      |
| ------------ | ---------------------------------- |
| 민감 데이터? | PII, 결제 정보, 비밀번호           |
| 입력 검증?   | XSS, SQL Injection, Path Traversal |
| 인증 필요?   | 엔드포인트별 인증 요구사항         |
| 권한 검사?   | 리소스 소유권, 역할 기반 접근      |
| 감사 로그?   | 누가, 언제, 무엇을                 |

### 에러 처리 관련

| 질문       | 세부 고려사항                           |
| ---------- | --------------------------------------- |
| 예상 에러? | 비즈니스 에러, 기술적 에러              |
| 에러 응답? | 사용자 메시지, 개발자 메시지, 에러 코드 |
| 에러 복구? | 재시도, fallback, graceful degradation  |
| 에러 알림? | 로깅, 모니터링, 알람                    |

### 테스트 관련

| 질문           | 세부 고려사항                       |
| -------------- | ----------------------------------- |
| 테스트 필요?   | 유닛, 통합, E2E                     |
| 테스트 범위?   | 핵심 로직, 엣지 케이스, 에러 케이스 |
| 테스트 도구?   | Jest, Vitest, Cypress, Playwright   |
| 테스트 데이터? | 목 데이터, 픽스처, 팩토리           |

## 상황별 추가 질문

### 기존 기능 수정 요청

```
• 현재 어떻게 동작하나요?
• 어떤 점이 문제인가요?
• 기대하는 동작은?
• 기존 사용자에게 영향?
• 하위 호환성 필요?
```

### 성능 개선 요청

```
• 현재 성능 측정값?
• 목표 성능?
• 병목 지점 파악?
• 트레이드오프 허용 범위?
```

### 버그 수정 요청

```
• 재현 단계?
• 예상 동작 vs 실제 동작?
• 발생 빈도?
• 영향 범위?
• 에러 메시지/로그?
```

### 리팩토링 요청

```
• 리팩토링 목적? (가독성/성능/확장성)
• 기능 변경 없이?
• 테스트 커버리지?
• 점진적/일괄 변경?
```

## 질문 순서 최적화

### 레벨 1: 필수 질문 (항상)

1. WHAT - 무엇을 만들 것인가
2. WHERE - 어디에 위치하는가
3. 기존 코드와의 관계

### 레벨 2: 상황별 질문 (맥락에 따라)

- 프론트엔드 → UI/UX 상세
- 백엔드 → API/데이터 상세
- 풀스택 → 데이터 흐름

### 레벨 3: 심층 질문 (필요 시)

- 엣지 케이스
- 성능/보안 요구사항
- 테스트 요구사항

### 레벨 4: 확인 질문 (마무리)

- 이해한 내용 정리
- 누락된 내용 확인
- 구현 계획 동의
